Características de la Plataforma Meteorológica

Aquí se encuentran la combinación de algunas tecnologías, que fueron utilizadas para el desarrollo del portal.

MQTT.

La comunicación con las estaciones es a través del protocolo, Message Queue Telemetry Transport (MQTT) es usado para la comunicación machine-to-machine (M2M) en el “Internet of Things“, este protocolo está orientado a la comunicación de sensores, debido a que consume muy poco ancho de banda y puede ser utilizado en la mayoría de los dispositivos empotrados con pocos recursos.

Java/AngularJS.

En diseño de software el front-end es la parte del software que interactúa con el o los usuarios y el back-end es la parte que procesa la entrada desde el front-end, Hemos utilizado Java como lenguaje de programacion para la parte de back-end y AngularJS que es un framework MVC de JavaScript para el Desarrollo Web front-end .

Open source.

Todo las herramientas que han sido utilizadas en este proyecto, es software distribuido y desarrollado libremente, son esenciales no solamente para el bien del usuario individual sino para la sociedad entera, estas permiten compartir y cooperar, algunas de estas herramientas son MongoDB, RabbitMQ, AngularJs y Java.

Seguridad.

Para implementar seguridad en nuestra aplicación, hemos utilizado el framework llamado Spring Security, este permite controlar desde el protocolo de seguridad, hasta los roles que necesitan los usuarios para acceder a los diferentes recursos de la aplicación, para poder acceder a los datos históricos debe crear una cuenta, que le permitirá visualizarlos .

Alertas.

La plataforma permite establecer dos tipos de Alertas, la primera es cuando una estación a dejado de enviar datos y la segunda cuando se ha establecido una condición (mayor, menor e igual) que supere un rango de una determinada variable metereologica.

Notificaciones.

Se pueden recibir notificaciones de alertas que ha sido establecidas, a través de correo electrónico y mensajes SMS.

Descargar Datos.

Se puede descargar el histórico de una determinada variable meteorológica, en un periodo de tiempo, estos datos los puedes obtener en dos tipos de formato CSV y JSON.

Características de las Estaciones Metereologicas

Las estaciones meteorológicas presentan las siguientes características.

Open Hardware

Todos los dispositivos de hardware que fueron utilizados, están basados en cuyas especificaciones y diagramas esquemáticos son de acceso público, ya sea bajo algún tipo de pago, o de forma gratuita, esto permite que el hardware pueda ser de calidad, de estándares abiertos y que sean más económicos.

Digimesh

En diseño de software el front-end es la parte del software que interactúa con el o los usuarios y el back-end es la parte que procesa la entrada desde el front-end, Hemos utilizado Java como lenguaje de programacion para la parte de back-end y AngularJS que es un framework MVC de JavaScript para el Desarrollo Web front-end .

Autonomia energetica

Todas las estaciones meteorológicas contienen un módulo, que le permite tener autonomía energética, esto es muy importante ya que todas ellas se encuentran en lugares apartados, donde no existe la posibilidad de conectar directamente a una corriente eléctrica.

Sensores Metereologicos

Las estaciones meteorológicas, tienen como ventaja que se puede conectar una variedad de sensores, lo que les permite adaptarse a la climatología del ambiente.
